# igame-china-journal

## 项目简介

igame-china-journal 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何单个域名或具体网站，仅作为页面内容的存储与展示用途。

## 目录说明

```
├── pages/          # 存放独立 HTML 页面文件
├── assets/         # 页面引用的资源文件（样式、脚本、图片等）
├── index.html      # 仓库首页入口（可选）
└── README.md       # 本文件
```

- `pages/` 目录下每个子文件夹或文件代表一个独立的归档页面。
- `assets/` 目录用于存放页面所需的公共或私有资源。
- 各页面之间相互独立，不依赖外部特定服务。

## 页面归档说明

- 所有页面均为静态 HTML 文件，无后端依赖。
- 页面内容可能包含文本、图片、样式与脚本，均为归档用途。
- 页面可能引用外部资源（如字体、库文件），但不依赖特定域名或网络环境。
- 归档页面按时间或主题分类存放，具体分类方式见 `pages/` 内文件结构。

## 维护说明

- 本仓库由维护者不定期更新，添加新的归档页面或修正已有页面。
- 欢迎通过 Issue 或 Pull Request 提交建议或修正。
- 提交新页面时请确保：
  - 页面为独立 HTML 文件，可离线查看。
  - 资源文件存放于 `assets/` 对应子目录。
  - 不包含恶意代码、跟踪脚本或侵犯他人权益的内容。
- 仓库不保证页面内容的持续可用性，但会尽力保持结构稳定。
